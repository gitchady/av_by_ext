# Plan: AV.by Currency Converter Extension

## Цель

Собрать легкое browser extension для `av.by`, которое показывает рядом с ценой в `BYN` эквивалент в `USD`, `EUR` или `RUB`, не ломается на динамическом контенте и не требует лишних разрешений.

## Базовые принципы

- `Manifest V3` и минимальные `host_permissions`.
- `Vanilla JS` без лишних зависимостей.
- Приоритет безопасности, устойчивости к изменениям DOM и простой поддержки.
- Курс по умолчанию берется из `NBRB`, но ручной курс из настроек имеет приоритет.
- Изменения настроек должны отражаться на странице без перезагрузки.

## Предлагаемая структура

- `manifest.json`:
  описание extension, permissions, content scripts, popup, background worker.
- `background.js`:
  загрузка и кэширование курсов, `alarms`, ответы на runtime messages.
- `content.js`:
  поиск цен на странице, sanitization, конвертация, DOM injection, `MutationObserver`.
- `popup.html`, `popup.js`, `popup.css`:
  управление валютой, источником курса и ручным значением.
- `storage.js` или `config.js`:
  единые ключи и helper-функции для настроек и кэша.
- `privacy-policy.html`:
  статичная privacy policy для публикации.
- `icons/`:
  иконки `16x16`, `48x48`, `128x128`.

## Этапы реализации

### 1. Архитектура и безопасность

- Создать `manifest.json` на `MV3`.
- Ограничить `host_permissions` до `*://*.av.by/*` и `*://*.nbrb.by/*`.
- Подключить `background service worker`.
- Подготовить каркас popup и policy page.

### 2. Data layer

- Реализовать запрос к `NBRB` API.
- Сохранять курс и timestamp в `chrome.storage.local`.
- Обновлять данные по alarm не чаще одного раза в 24 часа.
- Добавить логику приоритета:
  ручной курс из настроек выше API-курса.
- Подготовить единый интерфейс чтения курса для content script и popup.

### 3. Устойчивый DOM parsing

- Искать цены через комбинацию attribute selectors и regex, а не через хрупкие цепочки классов.
- Нормализовать строку вида `45 000 р.` в число.
- Исключить повторную обработку уже модифицированных узлов.
- Настроить `MutationObserver` для списков, карточек и бесконечного скролла.

### 4. UI и поведение на странице

- Вставлять конвертированную цену рядом с оригинальной ценой в `BYN`.
- Стараться наследовать типографику исходного элемента, чтобы вставка выглядела нативно.
- В popup добавить:
  выбор валюты `USD/EUR/RUB`,
  выбор источника курса `auto/manual`,
  поле ручного курса.
- Реализовать мгновенный пересчет на странице через `chrome.runtime.sendMessage`.

### 5. Финализация

- Подготовить иконки extension.
- Проверить работу на страницах:
  поиск,
  карточка автомобиля,
  личный кабинет.
- Проверить сценарии:
  первый запуск,
  пустой кэш,
  недоступность API,
  ручной курс,
  повторный рендер динамического DOM.
- Подготовить материалы для публикации в store.

## Definition of Done

- Extension корректно показывает сконвертированную цену рядом с `BYN`.
- Пользователь может переключать `USD`, `EUR`, `RUB`.
- Пользователь может включать ручной курс, и он реально переопределяет API.
- Курс кэшируется и не запрашивается на каждом открытии страницы.
- Динамически подгруженные объявления тоже обрабатываются.
- Разметка страницы не засоряется дубликатами цены.
- В проекте есть privacy policy и базовый набор store assets.

## Ограничения текущего MVP

- Не добавлять тяжелые frameworks без явной необходимости.
- Не расширять список permissions сверх roadmap без отдельного решения.
- Не включать roadmap v2 в MVP:
  `Remote Config`,
  tooltip с датой курса,
  калькулятор растаможки.

## Следующий практический порядок работ

1. Создать каркас extension-файлов и `manifest.json`.
2. Поднять `background.js` с кэшем курса.
3. Реализовать popup и сохранение настроек.
4. Добавить `content.js` с парсингом цен и injection.
5. Подключить `MutationObserver` и защиту от дубликатов.
6. Прогнать ручную проверку по основным типам страниц.
